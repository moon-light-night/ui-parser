# C4 Context Diagram - UI Screenshot Analyzer

C4 Context отображает систему целиком и ее связи.

## Диаграмма

```mermaid
C4Context
    title C4 Context - UI Screenshot Analyzer

    Person(user, "Пользователь", "Открывает веб-приложение в браузере. Загружает скриншоты, запускает анализ и ведёт диалог по содержимому экрана.")

    System_Ext(minio, "MinIO (S3)", "S3 объектное хранилище. Хранит загруженные файлы скриншотов. Поддерживает presigned URL для прямой загрузки файлов из браузера.")

    System_Boundary(boundary, "UI Screenshot Analyzer") {
        System(system, "UI Screenshot Analyzer", "Принимает скриншоты интерфейса, анализирует их с помощью VLM и поддерживает чат-сессии с контекстом конкретного экрана.")
    }

    System_Ext(ollama, "Ollama", "Локальный LLM / VLM сервер. Предоставляет мультимодальную модель для анализа изображений и генерации ответов в чате.")

    Rel(user, system, "Загружает скриншоты, запускает анализ, ведёт чат", "HTTPS · gRPC-Web")
    Rel(user, minio, "Загружает файл напрямую по presigned URL", "HTTPS PUT")
    Rel(system, minio, "Сохраняет и читает файлы скриншотов", "S3 API (HTTP)")
    Rel(system, ollama, "Отправляет изображения и промпты, получает структурированный JSON и потоковые ответы чата", "HTTP (Ollama REST API)")

    UpdateLayoutConfig($c4ShapeInRow="2", $c4BoundaryInRow="1")
```

## Участники

| Элемент                    | Тип             | Описание                                                                                                                       |
|----------------------------|-----------------|--------------------------------------------------------------------------------------------------------------------------------|
| **Пользователь**           | Person          | Единственный тип пользователя в MVP (аутентификация не реализована). Работает через браузер.                                   |
| **UI Screenshot Analyzer** | Software System | Весь сервис: фронтенд, API, анализатор, БД.                                                                                    |
| **Ollama**                 | External System | Запускается локально на хост-машине. Предоставляет HTTP API для работы с мультимодальными моделями.                            |
| **MinIO (S3)**             | External System | S3-совместимое хранилище. В локальной разработке - контейнер MinIO. В production заменяется на AWS S3 или другой S3-провайдер. |

## Ключевые архитектурные особенности

- **Прямая загрузка в S3** - браузер загружает файл напрямую в MinIO по presigned URL, минуя backend.
- **Ollama запускается локально** - модель работает на хост-машине, контейнеры обращаются к ней через `host.docker.internal:11434`. Интернет не требуется.
- **gRPC-Web через Envoy** - браузер не может использовать gRPC напрямую; Envoy транслирует gRPC-Web (HTTP/1.1) во внутренний gRPC (HTTP/2).
